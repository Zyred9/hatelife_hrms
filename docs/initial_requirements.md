# 初始录入需求 — 折耳根绩效管理平台

## 1. 需求背景

开发一个企业内部使用的绩效管理平台，企业名称：折耳根。

---

## 2. 需求范围

- 一套 H5 网页站点，网站名称：折耳根绩效管理平台
- 配套数据库

---

## 3. 后端架构概述

> 以下后台模块为前端网页提供服务，模块的划分可以微调修改，模块的名字存为标签便于沟通。

### 3.1 用户模块

- **管理员用户**：指定账号 `gongyuwen`、密码 `gongyuwen`，有所有数据权限、所有功能权限
- **普通用户关系**：多层、多对多上下级关系，直属上级可以管理下级
  - 管理员创建账号，建立上下级关系
  - 我有下级 → 身份（用户组）为"领导"；我没下级 → 身份为"员工"
- **用户基础信息**：姓名、性别、年龄、手机号、密码、部门

### 3.2 OKR 模块

- 管理每位员工的 OKR 档案

### 3.3 绩效评分模块

- 管理每位员工的绩效评分档案

### 3.4 基本功能模块

- 公共基础模块：图片上传、图片 OKR 识别、站内消息等
- 被具体业务功能调用

### 3.5 业务功能模块

- 糅合业务逻辑，例如"OKR 自评"，具体情况具体处理

### 3.6 助手模块

- 助手名称：根宝、多多、蛋挞（银渐层猫、布偶猫、金渐层猫的 3D 卡通形象）
- 问答常见问题，由管理员配置问题库
- 书评模块：联网检索书的信息，给到按章节的摘要、读书笔记

---

## 4. 前端网页需求描述

### 4.1 登录页

- 支持管理员、非管理员登录，登录后进入首页

### 4.2 首页

| 项目 | 描述 |
|------|------|
| 权限控制 | 无 |
| AI 助手 | 展示形象，支持三种形象切换；每隔 1 分钟展示关怀话术；搜索框支持功能搜索、问题库检索匹配 |
| 书架 | 展示历史解读的书，每天可新增解读一本书，输入完整书名开始解读 |
| 日历 | 有流程的日子加深标注 |
| 流程待办 | 当前需要处理的流程 + 历史流程 2 个 tab |
| 最近访问 | 最近访问的 5 个功能入口 |

### 4.3 我的 OKR

| 项目 | 描述 |
|------|------|
| 权限控制 | 无 |
| 列表展示 | 用户自己的 OKR，状态：①正常 ②审核中 ③驳回待修改 ④作废 |
| 状态按钮 | 正常：展示新增、编辑；审核中：不展示按钮；驳回待修改：展示编辑；作废：展示新增 |
| 新增 OKR | 手填 / 图片两种方式上传；新增后为审核状态，无法再次新增 |
| 编辑 OKR | 可编辑；编辑后为审核中状态，无法再次编辑 |
| 审核提交 | 1 个上级 → 上级收到待办；多个上级 → 同时审批，任一驳回则状态为驳回待修改；无上级 → 管理员审核 |
| 评分系统 | OKR 与评分一对一关系；包含自评、领导评分、最终评分（三方均可见） |

**评分流程**：
1. 正常状态 OKR → 自评
2. 自评后：有领导 → 领导评分；无领导 → 直接最终评分
3. 领导评分后 → 管理员录入最终评分
4. 评分维度见附件 2（评分规则）
5. 评分为表单，总分 = 每项小分 × 权重，自动计算
6. 评分完成后不可修改

### 4.4 团队 OKR

| 项目 | 描述 |
|------|------|
| 权限控制 | 管理员 或 当前我是领导（有下级） |
| 列表展示 | 管理员：按领导分栏展示所有人 OKR；领导：展示其下级 OKR |
| 审核操作 | 审核中状态可操作：①通过 ②驳回 ③作废 |
| 评分操作 | 根据评分阶段，录入领导评分或最终评分 |

### 4.5 账号管理

| 项目 | 描述 |
|------|------|
| 权限控制 | 仅管理员 |
| 功能 | 账号列表，支持增删改查 |

### 4.6 团队管理

| 项目 | 描述 |
|------|------|
| 权限控制 | 仅管理员 |
| 功能 | 展示画布，每部门一列，管理员可连线（单向箭头，箭头指向 = 上级） |

### 4.7 系统设置

| 项目 | 描述 |
|------|------|
| 权限控制 | 仅管理员 |
| 评分规则 | 调整员工、领导的考核周期、维度、权重、描述 |
| 消息提醒 | 开关配置：OKR 流程提醒、评分流程提醒、OKR 提交提醒（按用户组）、自评提醒（按用户组） |
